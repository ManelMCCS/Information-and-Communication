Camada de aplicação

Serviços TCP

O TCP é um serviço orientado à conexão e um serviço confiável de transferência de dados.

Serviço orientado a conexão

O TCP faz com que o cliente e o servidor troquem informações de controle na camada de transporte entre si antes das mensagens no nível do aplicativo começar a fluir. Este procedimento chamado de handshaking alerta o cliente e o servidor, permitindo que eles se preparem para a troca de pacotes. Após a fase de handshaking, diz-se que uma existe uma conexão TCP entre os sockets dos dois processos. Estabelece-se uma conexão full-duplex em que os dois processos podem enviar mensagens um ao outro pela conexão ao mesmo tempo. Quando o aplicativo terminar de enviar mensagens, deve interromper a conexão.

Serviço de transferência de dados

Os processos de comunicação podem contar com TCP para entregar todos os dados enviados sem erros e na ordem adequada. Quando um lado de o aplicativo passa um fluxo de bytes num socket, ele pode contar com o TCP para entregar o mesmo fluxo de bytes para o socket recetor, sem bytes em falta ou duplicados.

O TCP também inclui um mecanismo de controle de congestionamento, que estrangula um processo de envio (cliente ou servidor) quando a rede está congestionada entre o emissor e o receptor.

Segurança do TCP

Nem TCP nem o UDP fornecem qualquer criptografia - os dados que o processo de envio passa no seu socket são os mesmos dados que passam pela rede. Assim, por exemplo, se o processo de envio enviar uma senha em texto simples (ou seja, não criptografado) no seu socket, a senha de texto não criptografado percorrerá todos os links entre remetente e receptor, sendo possivelmente intreceptado por terceiros.

Como a privacidade e outras questões de segurança se tornaram críticas para muitos aplicativos, a comunidade da Internet desenvolveu um aprimoramento para o TCP, chamado Secure Sockets Layer (SSL). TCP aprimorado com SSL oferece serviços de segurança, incluindo criptografia, integridade de dados e autenticação de terminal.
